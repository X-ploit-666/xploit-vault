The Verbs (Methods)

The HTTP Method tells the server what action to perform on the resource.

GET

READ

Retrieves data. Parameters are in the URL. Safe and Cacheable.

POST

CREATE

Submits data to be processed. Data is in the Body. NOT Idempotent.

PUT

UPDATE

Replaces a resource entirely. Idempotent (Repeating it is safe).

DELETE

REMOVE

Removes the resource. Dangerous operation.

The Server's Reply: Status Codes

Every request gets a 3-digit number in response. These codes are grouped into five classes that tell the client exactly what happened.

2xx
Success
Request received, understood, and accepted. (e.g., 200 OK)
3xx
Redirection
Further action needed to complete request. (e.g., 301 Moved)
4xx
Client Error
Bad syntax or cannot be fulfilled. (e.g., 404 Not Found)
5xx
Server Error
Server failed to fulfill a valid request. (e.g., 500 Error)
