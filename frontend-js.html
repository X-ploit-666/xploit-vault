> 01_THE_BRAIN_OF_THE_WEB

JavaScript (JS) is one of the most popular programming languages on Earth. If HTML is the skeleton and CSS is the skin, JavaScript is the Brain. It is the part that makes things actually happen. Usually, it works on the "front end" (the part of a website you see in your browser). However, it can also be used on the "back end" (the hidden servers that run the whole app) using a tool called NodeJS.

Without JavaScript, a webpage would be "static," meaning it just sits there like a printed poster. JavaScript makes the page "dynamic" or interactive. It allows the page to change its look in real-time, accept your typing, process information, and update what you see without having to refresh the entire page. It is also used for advanced animations that CSS can't do alone and for sending secret messages to servers to get new data (a process called Ajax).

> LOADING_THE_ENGINE

To put JavaScript into a webpage, we use the <script> tag. You can either type the code directly inside these tags or tell the browser to go find a separate JavaScript file using a "src" (source) link, which looks like this: <script src="filename.js"></script>.

Every modern web browser has a built-in "JavaScript Engine." This means the browser itself does the thinking and runs the code right there on your computer (client-side), which makes the website feel very fast.

> 02_JS_FRAMEWORKS

As websites get more complicated, writing every single line of code from scratch becomes too slow. To fix this, developers created JavaScript Frameworks. These are collections of pre-written tools and "libraries" that make building big apps much easier. They help with things like letting users log in or creating pages that change instantly. Some of the most common toolkits are:

Framework Primary Use
AngularBuilding large, powerful corporate apps.
ReactCreating fast, smooth user interfaces.
VueA flexible, easy-to-learn toolkit for web apps.
jQueryAn older, classic tool for simple tasks.

> 03_CODE_ANALYSIS

Let's look at how JavaScript actually manipulates a webpage.

Example 1: The script tag

<script type="text/javascript"> ..JavaScript code.. </script>

This tells the browser: "The text between these tags is not a heading or a paragraph; it is a set of logical instructions (JavaScript) that you need to follow."

Example 2: External Scripts

<script src="./script.js"></script>

This tells the browser: "Don't look for the code here. Go find a separate file named script.js in this same folder and use the instructions inside it."

Example 3: Changing the page

document.getElementById("button1").innerHTML = "Changed Text!";
Part of Code What it means in plain English
document This refers to the entire webpage you are looking at.
.getElementById("button1") This tells the browser: "Look through the whole page and find the specific element that has the ID name badge 'button1'."
.innerHTML This refers to the actual text or content sitting inside that element.
= "Changed Text!" This is the command to swap the old text for the new words "Changed Text!".
