The DNS "Relay Race"

When you type a URL, a lightning-fast chain reaction occurs. Your request travels through a hierarchy of servers to find the correct address.

1. The Request

You type example.com. Your computer first checks its own local Cache (memory). If found, the race ends here!

2. The Resolver (The Helper)

If not cached, the request goes to your ISP's DNS Resolver. It acts as your personal assistant, doing the hard work of searching the internet for you.

3. The Root Server (The Librarian)

The Resolver asks the Root Server. It doesn't know the address, but it knows who manages .com domains.

4. The TLD Server (The Regional Map)

The Resolver is sent to the Top-Level Domain (TLD) server. This server points the way to the specific server managing example.com.

5. Authoritative Server (The Final Stop)

The request reaches the Authoritative Server. This is the master record holder. It replies with the exact IP: 192.0.2.1.

6. Connection Established

The Resolver hands the IP to your computer. Your browser connects to the website. The IP is cached for next time.

Why Cache is King

The "Relay Race" involves multiple servers across the globe. Doing this every single time you load a page would make the internet feel sluggish.

TTL (Time To Live): This setting in the DNS records tells your computer how long to keep the IP address in memory (cache) before asking again. High TTL means faster repeated visits.

The Language of DNS

A Zone File is like a database containing different types of "Records." Each record type serves a specific purpose in directing traffic.

Common Record Types

  • A Maps a hostname to an IPv4 address (Standard).
  • AAAA Maps a hostname to an IPv6 address (Newer, longer).
  • CNAME An alias. Points one name to another name.
  • MX Mail Exchange. Directs email to the mail server.

The Secret Shortcut: Hosts File

Your computer contains a manual override file. If an entry exists here, the entire DNS relay race is skipped. It's often used for blocking sites or web development testing.

/etc/hosts - Administrator
# This is a comment. Format: IP Address Hostname
127.0.0.1 localhost # My computer
192.168.1.10 dev-server.local # Development work
0.0.0.0 unwanted-site.com # BLOCKED SITE
