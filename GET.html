The web isn't just about viewing pages; it's a constant conversation. Understanding how browsers ask for resources via GET and how servers protect them via Basic Auth is the first step in web security mastery.

The Anatomy of a GET Request

When you perform a search, the browser sends your data directly in the URL. This is known as a Query Parameter. It allows data to be bookmarked and shared, but it also makes it visible to anyone watching the network traffic.

The Server Gatekeeper

HTTP Basic Auth is a primitive access control method. Unlike a login form, it's handled by the web server (like Apache or Nginx). It forces the browser to show a native popup asking for credentials before loading any page content.

The Handshake

  • 401 The server blocks the first request and demands credentials via the WWW-Authenticate header.
  • 200 Client resends request with an Authorization header containing the encoded credentials.

⚠ The Base64 Security Myth

Basic Auth transmits credentials in the header formatted as admin:admin. However, to travel safely over HTTP, this string is encoded into Base64. Encoding is NOT Encryption.

# The Transformation

Plaintext
admin:admin
➝
Base64
YWRtaW46YWRtaW4=
$ echo "YWRtaW46YWRtaW4=" | base64 -d
> admin:admin

Terminal Mastery: cURL

Security professionals bypass the browser UI to interact directly with the server using cURL. Here are the essential flags for handling Basic Auth.

-u flag

Automatic Header Generation

curl -u admin:admin http://target.com

Simply takes user:pass and handles the Base64 encoding for you.

Inline Auth

URL Embedded Credentials

curl http://admin:admin@target.com

Browsers also support this format for quick access.

-H flag

Manual Header Injection

curl -H 'Authorization: Basic YWR...'

Useful for replaying captured credentials (Pass-the-Hash style).
