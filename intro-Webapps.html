Evolution of Digital Interaction

Web 1.0 vs. Web 2.0 vs. Native

We have moved from static digital brochures to fully functional applications running in the browser. Web Applications (Web 2.0) combine the accessibility of websites with the interactivity of native software.

  • 1 Web 1.0: Static content. Hard to update. No interaction.
  • 2 Native Apps: Fast, hardware access, but require installation and manual updates.
  • 3 Web Apps: Instant updates, accessible anywhere, zero installation.

Infrastructure Models

How servers and databases are arranged determines the security, reliability, and scalability of an application.

1. The "All-in-One" (Risky)

App + DB + OS

Everything lives on one machine. If the server is hacked or crashes, everything is lost.

2. Separated Database

DB

Web Server and Database are separate. If the web server is compromised, the database has an extra layer of protection.

3. Multi-Server & Redundancy

Primary
Backup

Uses Load Balancers and Backup Databases. High availability and strong security segmentation.

The Full Stack Ecosystem

A web application is divided into the Front End (what you see) and the Back End (what processes data).

Front End (The Trinity)

Runs in the browser. Must be responsive for all devices.

HTML (Structure) CSS (Style) JS (Logic)

Back End (The Engine)

Runs on the server. Handles authentication, databases, and business logic.

Includes: Web Servers (Apache/NGINX), Databases (SQL/NoSQL), and Logic Frameworks.

The Danger Zone: Security Risks

Web apps are global targets. Attackers look for specific vulnerabilities to steal data or seize control.

OWASP Top Vulnerabilities & Impact

SQL Injection (SQLi)

Attackers inject malicious commands into input fields to trick the database into revealing hidden secrets.

IDOR

Insecure Direct Object Reference. Changing a URL ID (e.g., /user/5 to /user/6) to view someone else's profile.

Broken Access Control

The #1 vulnerability. Regular users finding ways to access Admin functions due to poor design.
