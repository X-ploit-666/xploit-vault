Identifying Software Threats Before They Happen

STRIDE is a powerful mnemonic created by Loren Kohnfelder and Praerit Garg to help developers perform threat analysis. By understanding the six specific ways software can be attacked, you can build stronger, more resilient systems.

The 6 Pillars of Vulnerability

A balanced view of the core threat categories that put systems at risk.

Detailed Threat Breakdown

Every threat type violates a specific security goal. Explore the definitions and examples below.

🎭

Spoofing

Breaks: Authentication

Pretending to be someone or something else, like a famous person, a specific website, or a system file.

Example: Sending an email that looks like it is from your bank.
✏️

Tampering

Breaks: Integrity

Changing information on a disk, a network, or inside a computer's memory without permission.

Example: Changing the numbers in a spreadsheet or database.
🤷

Repudiation

Breaks: Non-repudiation

Claiming you didn't perform an action after doing it, effectively denying responsibility.

Example: A user claiming they never ordered an expensive item.
🔓

Info Disclosure

Breaks: Confidentiality

Leaking secrets or showing information to people who aren't allowed to see it.

Example: A computer showing your password in an error message.
💥

Denial of Service

Breaks: Availability

Using up a computer's resources so the system stops working for everyone else.

Example: Flooding a website with fake traffic so real users can't get in.
👑

Elevation of Privilege

Breaks: Authorization

A regular user finding a way to get the powers of an administrator or "boss" user.

Example: Exploiting a bug to delete other people's files.

Case Study: Denial of Service

Analyzing a classic "Fork Bomb" attack.

The Logic of Exhaustion

A Denial of Service (DoS) attack doesn't always require massive external networks. Sometimes, a single line of malicious code is enough to cripple a system by forcing it to consume all its own resources.

while(1){
  fork();
}
1

The Infinite Loop

while(1) forces the computer to execute the enclosed command forever, without a stop condition.

2

The Clone Command

fork() tells the system to duplicate the current process. One becomes two, two become four, scaling exponentially until the CPU and memory freeze.
