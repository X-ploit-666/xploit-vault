🗝️ OAuth Access Paths

Grant Types & Scopes

If OAuth is the "Hotel Key Card," then Grant Types are the different ways you can check into the hotel, and Scopes are the specific floors your card is allowed to reach.

🛤️

Grant Type = "The Route"

This defines the exact sequence of steps the app takes to get the key. It is the how.

Examples: Authorization Code, Implicit
🛡️

Scope = "The Permissions"

This defines exactly what data the app can touch once it has the key. It is the what.

  • • profile (read name)
  • • email (view address)
  • • contacts.read (see list but not edit)

The Two Main Paths

Comparing the secure back-office exchange with the risky public handoff.

Path 1: Authorization Code

The "Secure Desk" Flow

You don't get the key directly. You get a "pickup slip" first, exchanged in private.

1. App asks for access
⬇️
2. You log in and say "Yes"
⬇️
3. App gets a temporary code (Pickup Slip)
⬇️
4. App's server sends code + secret password behind the scenes
⬇️
5. App gets Access Token

✔️ Verdict: Safe. The actual key is never seen by your web browser. Exchange happens behind the scenes.

Path 2: Implicit

The "Direct Delivery" Flow

Skip the middleman. The key is handed directly to the browser.

1. App asks for access
⬇️
2. You log in and say "Yes"
⬇️
3. Website sends Access Token directly in URL bar
(No server exchange steps)

❌ Verdict: Risky. The key is visible in browser history. Anyone nearby or a malicious script could grab it.

Security & Architecture Analysis

When analyzing the two flows, the Authorization Code grant offers significantly higher security because it leverages a backend server to hold the client secret.

The Implicit flow was originally designed for early Single Page Applications (SPAs) that lacked a backend. However, because it exposes the token directly to the frontend client, its exposure risk is critically high by modern standards.

Analogy Check

Implicit flow is like handing someone a highly sensitive package in the middle of a crowded public lobby. Authorization code is verifying ID in a secure back office before handover.

Real-World Comparison

Feature Authorization Code Implicit
Best For Apps with a backend (Websites) Apps with no backend (Simple SPAs)
Security High (Secret exchange) Low (Exposed in URL)
The "Key" Token stays securely on the server Token sits exposed in the browser
Typical Use "Login with Google" on Airbnb Old-school single-page apps (Legacy)

Modern Industry Adoption

Due to security flaws, Implicit flow is deprecated in modern OAuth 2.1 specs.

💡

The Ultimate Takeaway

Grant Type is the path you take to get permission. Scope is what that permission allows you to do. If you have a choice, always take the Authorization Code path—it's like using a secure vault instead of an open counter.
