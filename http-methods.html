Understanding the fundamental language of the web. From Request Methods to Response Codes, this is how clients and servers communicate in the cyber domain.

1. THE ACTION: REQUEST METHODS

cyber_operation: EXECUTE

HTTP methods are the "verbs" of the web. They tell the server exactly what cyber operation to perform. In REST API architecture, selecting the correct verb is critical for system stability and semantic correctness.

GET READ

Requests data. Parameters visible in URL. Safe & Idempotent.

POST CREATE

Sends data in body. Used for logins/uploads. NOT Idempotent.

DELETE REVISE

Removes a resource. Danger zone operation.

2. THE RESULT: STATUS CODES

The server responds with a 3-digit code to indicate the outcome. Understanding these classes is vital for debugging and user experience.

[2xx] SUCCESS

"System Green. Request Executed."

  • > 200 OK
  • > 201 Created
  • > 204 No Content

[3xx] REDIRECTION

"The file is in another castle."

  • > 301 Moved Permanently
  • > 302 Found (Temporary)

[4xx] CLIENT ERROR

"YOU messed up. Check request."

  • > 401 Unauthorized
  • > 403 Forbidden
  • > 404 Not Found

[5xx] SERVER ERROR

"I messed up. Circuits fried."

  • > 500 Internal Error
  • > 503 Service Unavailable

An operation is Idempotent if performing it multiple times has the same effect as doing it once.

GET / PUT / DELETE SAFE TO REPEAT
POST NOT IDEMPOTENT

Example: Posting a comment twice creates two duplicates. Deleting a file twice just ensures it's gone.
