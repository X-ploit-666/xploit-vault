A **Database** is a digital storage system that lives on the back-end server. It holds everything a web app needs to remember, like images, user posts, usernames, and passwords. This allows the app to show "dynamic content," meaning the website changes based on who is logged in.

When choosing a database, developers look for:

- **Speed:** How fast it can find or save info.
    
- **Size:** How much data it can hold.
    
- **Scalability:** How easily it grows as the website gets more popular.
    
- **Cost:** Whether it is free or expensive.
    

### 📊 1. Relational Databases (SQL)
![[web_apps_relational_db.jpg]]
Think of an **SQL Database** like a giant Excel workbook with multiple sheets. Everything is organized into **Tables**, **Rows**, and **Columns**.

- **Key:** A unique ID (like a Social Security number) assigned to a row.
    
- **Relationship:** Connecting two tables using these IDs. For example, a "Users" table links to a "Posts" table using a `user_id`. This saves space because you don't have to write the user's name on every single post.
    
- **Schema:** The master "blueprint" or "map" that shows how all these tables are linked together.
    

**Common SQL Databases:**

- **MySQL:** The world's most popular, free, and open-source.
    
- **MSSQL:** Microsoft's version, built for Windows servers.
    
- **Oracle:** Super powerful and reliable for huge banks/companies, but very expensive.
    
- **PostgreSQL:** Free and great for adding custom, advanced features.
    

### 📁 2. Non-Relational Databases (NoSQL)
![[web_apps_non-relational_db (1).jpg]]
**NoSQL** doesn't use the strict table-and-row structure. It is like a messy pile of folders. It is very flexible and "Scalable" (easy to expand).

**Common Storage Models:**

1. **Key-Value:** Like a dictionary where a label (the key) points to a piece of data (the value).
    
2. **Document-Based:** Stores data in complex **JSON** files (a way of writing data that looks like a list of properties).
    
3. **Wide-Column:** Optimized for massive amounts of data.
    
4. **Graph:** Used to map connections, like how friends are linked on Facebook.
    

**Common NoSQL Databases:**

- **MongoDB:** The most popular. Uses the Document-Based model.
    
- **ElasticSearch:** Built specifically for searching through giant piles of data very fast.
    

### 💻 Code Analysis: Database Interaction

The text shows how a programming language (like PHP) talks to a database (like MySQL).

**Step 1: Making the Connection**

```
// "mysqli" is a tool that opens a door to the database.
// It needs the server address ("localhost"), username, and password.
$conn = new mysqli("localhost", "user", "pass");
```

**Step 2: Creating and Selecting**

```
// This tells the database to create a new folder called "database1"
$sql = "CREATE DATABASE database1";
$conn->query($sql);

// Now we connect specifically to that new database to use it
$conn = new mysqli("localhost", "user", "pass", "database1");
```

**Step 3: Searching (The Query)**

```
// 1. Get what the user typed into a search box
$searchInput = $_POST['findUser'];

// 2. Write a "Query" (a question for the database)
// "select * from users" means "Get everything from the user table"
// "where name like '%$searchInput%'" means "where the name matches what they typed"
$query = "select * from users where name like '%$searchInput%'";

// 3. Ask the database the question
$result = $conn->query($query);
```

**Step 4: Showing the Results**

```
// This loop goes through every row found and prints the "name" to the screen
while($row = $result->fetch_assoc() ){
    echo $row["name"]."<br>";
}
```

⚠️ **Hacker Note:** The text mentions **SQL Injection**. This happens when a hacker puts their own database commands into the `$searchInput` box to trick the server into deleting data or leaking passwords!
