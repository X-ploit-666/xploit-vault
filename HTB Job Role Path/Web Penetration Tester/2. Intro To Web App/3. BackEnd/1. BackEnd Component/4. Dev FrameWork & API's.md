### 🏗️ 1. Development Frameworks

Building a website from scratch is hard. A **Framework** is a pre-made "toolbox" or "blueprint" that handles the boring, repetitive parts (like user login or database connection) so developers can focus on the cool features.

**The Top Tools:**

- **Laravel (PHP):** Powerful and easy. Perfect for startups.
    
- **Express (Node.JS):** High-speed. Used by **Uber** and **PayPal**.
    
- **Django (Python):** Solid and secure. Used by **Instagram** and **YouTube**.
    
- **Rails (Ruby):** Great for building fast. Used by **GitHub** and **Airbnb**.
    

### 📡 2. APIs (The "Secret Handshake")
![[api_examples.jpg]]
An **API (Application Programming Interface)** is how the Front End (what you see) asks the Back End (the brain) to do something. It is a set of rules for how two programs talk to each other.

#### 🔎 Query Parameters

This is the most basic way to send info. You put the data directly into the web request.

- **GET Request:** The data is visible in the URL: `search.php?item=apples`.
    
- **POST Request:** The data is hidden inside the request body (better for passwords).
    

**Code Extract (HTTP POST Request):**

```
POST /search.php HTTP/1.1
...[Headers go here]...

item=apples
```

- **What it does:** This tells the server, "Go to the search page and look for 'apples'." The server sees `item=apples` at the very bottom and knows exactly what you want.
    

### 📜 3. API Standards: SOAP vs. REST

#### **SOAP (Simple Objects Access)**

Uses **XML** to send data. It is very formal and strict.

- **Best for:** Complex data or "stateful" apps (apps that need to remember exactly what you were doing).
    
- **The Downside:** It’s "heavy" and hard for beginners because the messages are long.
    

**Code Extract (SOAP Message):**

```
<?xml version="1.0"?>
<soap:Envelope xmlns:soap="[http://www.example.com/soap/](http://www.example.com/soap/)">
  <soap:Header></soap:Header>
  <soap:Body>
    <!-- Your data goes inside these tags -->
  </soap:Body>
</soap:Envelope>
```

- **What it does:** The `<soap:Envelope>` is like a digital shipping box. Everything inside the `<soap:Body>` is the actual "cargo" (the data) being sent.
    

#### **REST (Representational State Transfer)**

The modern favorite. It is "modular" and uses the URL path to find things (like `/users/1`).

- **Format:** Usually returns data in **JSON** (a simple list of keys and values).
    
- **Methods:**
    
    - `GET`: Read data.
        
    - `POST`: Create new data.
        
    - `PUT`: Update/Replace data.
        
    - `DELETE`: Trash data.
        

**Code Extract (JSON Response):**

```
{
  "100001": {
    "date": "01-01-2021",
    "content": "Welcome!"
  }
}
```

- **What it does:** This is the server's answer. It says, "For ID 100001, here is the date and the message." It is very easy for the Front End to read and show on your screen.
