A **Web Server** is a program that runs on the back-end machine. Its job is to handle "HTTP traffic"—which is just the data sent back and forth when you browse the web.
![[web-server-requests.jpg]]
### 🚦 How It Works (The Workflow)

Think of a web server like a receptionist at a massive office building.

1. **Request:** You (the client) ask for a page (like `index.html`).
    
2. **Routing:** The server finds that file or starts a process to create it.
    
3. **Response:** The server sends the file back to you with a "Status Code."
    

Web servers usually "listen" on specific **Ports**. A port is like a specific door on the computer:

- **Port 80:** Used for standard, unencrypted web traffic (HTTP).
    
- **Port 443:** Used for secure, encrypted web traffic (HTTPS).
    

### 🔢 Status Codes (The Server's Language)

When a server talks back, it uses numbers to tell you what happened:

|Category|Code|Meaning|
|---|---|---|
|**Success**|`200 OK`|Everything worked! Here is your page.|
|**Redirect**|`301/302`|The page moved. Go to this new address.|
|**Client Error**|`400`|The server didn't understand your request.|
|**Client Error**|`401/403`|You aren't logged in or you aren't allowed here.|
|**Client Error**|`404`|I can't find that page. It doesn't exist.|
|**Server Error**|`500`|The server crashed or hit a bug it couldn't fix.|
|**Server Error**|`502/504`|The server is waiting on another computer and timed out.|

### 💻 Code Analysis: cURL

The text shows how to use a tool called **cURL**. This is a command-line tool used to "talk" to servers without using a browser.

**Example 1: Getting "Headers"**

```
# -I means "Head Only" (Don't show the page, just the status)
curl -I [https://academy.hackthebox.com](https://academy.hackthebox.com)
```

- **What it does:** It asks the server for its "ID card."
    
- **Result:** It shows `HTTP/2 200`, meaning the connection is successful. It also shows the date and the type of content (`text/html`).
    

**Example 2: Getting the Source Code**

```
# Just the URL gives you the raw HTML code
curl [https://academy.hackthebox.com](https://academy.hackthebox.com)
```

- **What it does:** It pulls down the actual "skeleton" of the website.
    
- **Result:** You see tags like `<html>`, `<head>`, and `<title>`. This is what your browser reads to draw the website on your screen.
    

### 🏗️ The Big Three Web Servers

#### 1. Apache (The Classic)
![[apache.png]]
- **Market Share:** Hosts over 40% of the web.
    
- **Vibe:** Very flexible. You can add "Modules" (like plug-ins) to make it do more things.
    
- **Best For:** Beginners and PHP developers.
    
- **Users:** Apple, Adobe.
    

#### 2. NGINX (The Speedster)
![[nginx.png]]
- **Market Share:** Hosts about 30% of the web (60% of the top 100k sites).
    
- **Vibe:** Built for speed. It uses "Async" architecture, which means it can handle thousands of visitors at once without using much memory.
    
- **Best For:** High-traffic sites.
    
- **Users:** Google, Facebook, Netflix.
    

#### 3. IIS (The Corporate Choice)
![[iis.png]]
- **Market Share:** About 15% of the web.
    
- **Vibe:** Made by Microsoft. It works perfectly with Windows and "Active Directory" (the system companies use to manage employee logins).
    
- **Best For:** Windows-based businesses and `.NET` apps.
    
- **Users:** Microsoft, Stack Overflow.
