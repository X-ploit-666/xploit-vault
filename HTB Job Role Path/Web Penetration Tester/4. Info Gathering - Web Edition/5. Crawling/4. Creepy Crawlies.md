### 📝 The Simple Breakdown

Web crawling is a big and complicated job. It means using a computer program to visit websites and look at everything on them. You do not have to do this work by yourself. There are many digital tools that can help you. These tools are like robots that explore the web automatically. They make the work faster and better. This lets you spend your time looking at the information they find instead of doing the hard work of searching.

**Common Tools for the Job:**

- **Burp Suite Spider:** This is a part of a famous security testing app. Its "Spider" tool maps out how a website is built. It finds hidden pages and points out weak spots that might be unsafe.
    
- **OWASP ZAP:** This is a free tool that anyone can use to check a website's safety. It can search for weaknesses automatically or let you do it by hand.
    
- **Scrapy:** This is a set of building blocks for the Python computer language. It helps people build their own custom crawling robots. It is very flexible and can handle difficult websites.
    
- **Apache Nutch:** This is a very powerful tool made with Java. It is used for massive searches across the whole internet. It is harder to set up, so you need to be an expert to use it.
    

**Rules for Being a Good Crawler:** It is very important to be polite and follow the law. You must ask for permission before you crawl a website, especially if you are going to do a deep search. Do not send too many requests at once. If you do, you might make the website's computer (the **server**) slow down or crash.

### 💻 Code Breakdown

In this guide, we use specific commands to set up our tools. Here is what they do:

#### 1. Installing the Tool

```
pip3 install scrapy
```

- **What it does:** This uses `pip3` (a tool that gets Python software) to download and install **Scrapy** onto your computer.
    
- **Why it matters:** Without this, your computer won't understand how to run the crawling scripts.
    

#### 2. Getting the Custom Spider

```bash
wget -O ReconSpider.zip https://academy.hackthebox.com/storage/modules/144/ReconSpider.v1.2.zip
unzip ReconSpider.zip
```

- **wget:** This command downloads a file from the internet. The `-O` tells it to save the file with the name "ReconSpider.zip".
    
- **unzip:** This opens the compressed folder so you can use the files inside.
    
- **ReconSpider:** This is a "spider"—a specific script designed to crawl a target site (like a digital bloodhound).
    

#### 3. Running the Search

```
python3 ReconSpider.py http://inlanefreight.com
```

- **python3:** This tells your computer to run a Python program.
    
- **ReconSpider.py:** This is the specific script we just downloaded.
    
- **URL:** This is the website the spider will visit to collect data.

### 📁 Understanding the Results (`results.json`)

The tool saves everything it finds in a **JSON file**. Think of a JSON file as a digital filing cabinet where information is organized by "keys" (labels).

|Label (Key)|What it stores (Simple Terms)|
|---|---|
|**emails**|Any email addresses found on the site.|
|**links**|A list of other web pages the site points to.|
|**external_files**|Links to files like PDFs or Word docs.|
|**js_files**|JavaScript files (the code that makes the site interactive).|
|**form_fields**|Places where users type info (like login boxes).|
|**images**|The web addresses of all pictures on the site.|
|**comments**|Notes left by developers in the website's hidden code.|

By looking at these results, you can see how a website is built and find interesting things to study further.
