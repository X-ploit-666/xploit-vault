### What is Subdomain Brute-Forcing?

Subdomain brute-forcing is a way to find hidden parts of a website. Imagine a main website like `google.com`. It might have hidden sections like `mail.google.com` or `dev.google.com`.

**"Brute-forcing"** means trying a huge list of possible names one by one until you find ones that actually work. It is like trying every key on a massive keychain to see which ones open a door.

### How the Process Works (Step-by-Step)

1. **Wordlist Selection (Picking Your Guesses):** First, you need a "wordlist." This is just a text file full of common names like "blog," "test," or "admin."
    
    - **General lists:** Have very common names used by everyone.
        
    - **Targeted lists:** Use names specific to a certain business (like "banking" words for a bank).
        
    - **Custom lists:** Lists you make yourself based on what you already know about the target.
        
2. **Iteration (Combining Names):** A computer program takes every word from your list and attaches it to the main website name.
    
    - _Example:_ If the list has "dev" and the site is "example.com," the tool creates `dev.example.com`.
        
3. **DNS Lookup (Checking if it’s Real):** The tool asks the internet's "phonebook" (the **DNS** or Domain Name System) if that new name exists. It looks for **IP addresses** (the digital address of a computer) using:
    
    - **A Records:** For standard internet addresses.
        
    - **AAAA Records:** For the newer version of internet addresses.
        
4. **Validation (Saving the Results):** If the internet says "Yes, that address exists," the tool saves it. You can then check it in a browser to see what is actually there.
    

### Popular Tools Used

- **dnsenum:** A big toolkit that does many jobs at once.
    
- **fierce:** Good for finding "folders within folders" (recursion).
    
- **dnsrecon:** A flexible tool that gathers data in many different ways.
    
- **amass:** Very powerful and connects with other data sources.
    
- **assetfinder:** A fast, simple tool for quick checks.
    
- **puredns:** Great at handling massive lists of names very quickly.
    

### Deep Dive: Using the `dnsenum` Tool

`dnsenum` is a famous tool written in a language called **Perl**. It doesn't just guess subdomains; it also:

- **Lists Records:** Shows the technical setup of a site.
    
- **Zone Transfers:** Tries to trick the site into giving away its whole map (usually blocked, but a big win if it works).
    
- **Google Scraping:** Searches Google to see what names are already public.
    
- **Reverse Lookup:** Takes an address and asks, "What website lives here?"
    
- **WHOIS:** Finds out who owns the website.
    

### 💻 Code Breakdown

Here is the command used in the example:

```
dnsenum --enum inlanefreight.com -f /usr/share/seclists/Discovery/DNS/subdomains-top1million-20000.txt -r
```

**What each part means:**

- `dnsenum`: This starts the program.
    
- `--enum inlanefreight.com`: This tells the tool the target website is `inlanefreight.com`. The `--enum` part turns on several helpful settings at once to gather information.
    
- `-f [path to file]`: The `-f` stands for **file**. This tells the tool where to find the wordlist (in this case, a list of the 20,000 most common subdomains).
    
- `-r`: This stands for **recursion**. If the tool finds `dev.inlanefreight.com`, it will then start guessing names for that new site too (like `test.dev.inlanefreight.com`).

```
[ WORDLIST ]  --->  [ THE TOOL ]  --->  [ THE INTERNET (DNS) ]
(List of names)    (dnsenum script)     (Is "dev.site.com" real?)
      |                   |                       |
      |                   |                       |
      +-------+-----------+                       |
              |                                   |
              V                                   V
      [ BUILDING NAMES ]                  [ CHECKING RESULTS ]
      1. dev + site.com                   If YES -> Save to list!
      2. blog + site.com                  If NO  -> Ignore it.
      3. mail + site.com
```

**Simplified Summary:**

1. **Pick your guesses** (Wordlist).
    
2. **Glue them to the domain** (Iteration).
    
3. **Ask the internet if they work** (DNS Query).
    
4. **Keep the winners** (Validation).
