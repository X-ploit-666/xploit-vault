### What are Subdomains?

When we talk about the internet, we usually look at the main name of a website, like `example.com`. This is called the **Main Domain**. But many websites have extra parts added to the beginning, like `blog.example.com` or `shop.example.com`. These are called **Subdomains**.

Think of the main domain like a large building. Subdomains are like different rooms or offices inside that building. Companies use them to keep different parts of their business separate. For example:

- `mail.example.com` is just for email.
    
- `dev.example.com` might be a place where programmers test new code.
    

### Why Do Security Researchers (Web Reconnaissance) Look for Them?

**Web Reconnaissance** is the act of gathering information about a target website to find weak spots. Looking for subdomains is important because they often hide things that aren't on the main page.

1. **Testing Areas (Development and Staging):** Before a company puts a new feature on their main site, they test it on a subdomain. These areas often have lower security, which makes them easier to break into. They might also leak secret information by accident.
    
2. **Secret Login Pages:** Some subdomains have "Admin Panels" or login doors meant only for employees. If a hacker finds these, they can try to guess passwords to get inside.
    
3. **Old Apps (Legacy Applications):** Sometimes companies forget about old websites they built years ago. These "Legacy" apps use old software that hasn't been updated, making them very easy to hack.
    
4. **Sensitive Data:** Sometimes, files that should be private (like internal documents or setup files) are accidentally left on a subdomain where anyone can find them.
    

### Finding Subdomains (Subdomain Enumeration)

**Subdomain Enumeration** is just a fancy way of saying "making a big list of every subdomain a company owns."

In the world of **DNS** (the "Phonebook" of the internet), these subdomains are found in two types of records:

- **A Records:** These link a name (like `blog.example.com`) to a computer's address (IP address).
    
- **CNAME Records:** These are "nicknames" that point one name to another name.
    

There are two ways to find these:

#### 1. Active Search (Active Enumeration)

This means you talk directly to the company's computer servers to ask what subdomains they have.

- **Zone Transfer:** This is like asking the server to "give me the whole phonebook." Most servers are locked so you can't do this, but if a server is set up wrong, it might leak everything.
    
- **Brute-Force:** This is when you use a computer program to guess names. The program tries thousands of words like `test.example.com`, `admin.example.com`, etc., to see which ones actually exist.
    
- **Tools:** Common tools for this are `dnsenum`, `ffuf`, and `gobuster`.
    

#### 2. Passive Search (Passive Enumeration)

This is a "sneaky" way to find subdomains without the company knowing you are looking. You look at public records instead of asking the company directly.

- **Certificate Logs:** When a site uses a "Lock" icon (SSL/TLS certificate) for security, the name is recorded in a public list called **Certificate Transparency (CT) logs**. You can look at these logs to see every subdomain name registered.
    
- **Search Engines:** You can use Google or DuckDuckGo. By typing `site:example.com`, the search engine will show you every page it knows about, including subdomains.
    
- **Databases:** Some websites collect internet data and let you search through it for free.
    

### Which is better?

- **Active** gives you more details but the company might notice you.
    
- **Passive** is quiet (stealthy) but might miss some hidden rooms. **The best strategy is to use both.**
