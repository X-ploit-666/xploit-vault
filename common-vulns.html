Where do vulnerabilities come from?

When security experts perform a Penetration Test, they are looking for weaknesses that attackers can exploit. These security gaps generally originate from two main sources.

1

Handmade Sites

Custom code written by developers often contains unique bugs or logical errors.

2

Code Breakdown: SQL Injection

Let's look at a vulnerable line of PHP code used to search for users. If the $searchInput is not filtered, the logic breaks.

The Danger Zone

If a user inputs ' OR '1'='1, the database sees True for every row, revealing all user data.

search.php
$query = "select * from users where name like '%$searchInput%'";
Code Part Meaning
$query The "box" holding the command for the database.
select * Grab every single piece of information.
where name like Only show people whose names match the input.
%$searchInput% The Vulnerability. This is exactly what the user typed.
