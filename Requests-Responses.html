The Anatomy of
The Request

The internet relies on a constant conversation between Clients (your browser) and Servers. This infographic decodes the HTTP protocol, visualizing how requests are formed, how servers respond, and how encryption shields this dialogue from prying eyes.

1. The Conversation

An HTTP interaction is binary: a Request (the ask) and a Response (the answer). The Request carries the Method (Verb) and Headers, while the Response carries the Status Code and Body.

OUTGOING
// THE REQUEST
GET /users/login.html HTTP/1.1
Host: inlanefreight.com
User-Agent: curl/7.68.0
INCOMING
// THE RESPONSE
HTTP/1.1 200 OK
Content-Type: text/html
<html>... (Body) ...</html>

Status Codes: The Verdict

Every response carries a code indicating success or failure.

200 OK Success
401 Auth Unauthorized
301 Move Redirect

3. The Digital Shield (HTTPS)

HTTP sends data in Clear-Text, making it vulnerable to Man-in-the-Middle (MiTM) attacks. HTTPS solves this by wrapping the connection in an encrypted SSL/TLS tunnel using Port 443.
