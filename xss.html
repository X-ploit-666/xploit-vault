> 01_BEYOND_INJECTION

Cross-Site Scripting (XSS) is like the evolved, more dangerous version of HTML Injection. While HTML Injection lets a hacker change the look of a page, XSS lets them inject and execute JavaScript code. Since JavaScript is the "brain" of the website, giving a hacker the power to run it means giving them control over what the website does for the victim.

Once a hacker can run code on your computer through a vulnerable website, they can steal your Cookies (the digital keys you use to stay logged in), hijack your account, or even try to take over your machine.

> THE_THREE_VARIANTS

Security experts divide XSS into three main categories based on how the malicious code reaches the victim:

Type In Simple Terms
Reflected XSS The hacker sends a link with the code hidden inside. When the victim clicks it, the website "reflects" the code back and runs it immediately.
Stored XSS The hacker saves the code on the website (like in a comment). Anyone who visits that page later will automatically run the code.
DOM XSS The code is executed entirely inside the browser's own map of the page (the DOM), usually by manipulating the URL or a username field.

> 02_THE_MALICIOUS_PAYLOAD

Using the same vulnerable name prompt from before, a hacker could enter this specialized script:

#">
Snippet Part The Hacker's Logic
#"> This "breaks" the current HTML tag so the hacker can start a new one.
<img src=/ ...> This tries to load a fake image. Since the source / is not a real image, it will fail.
onerror=... This is a backup command. It tells the browser: "If the image fails to load, run this JavaScript instead."
alert(document.cookie) The final blow. This pops up a window showing the user's secret login cookies. In a real attack, the hacker would send these cookies to their own server instead of showing an alert.
