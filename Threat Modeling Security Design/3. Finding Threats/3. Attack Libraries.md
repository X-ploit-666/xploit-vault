# Attack Libraries: A Simple Guide

Security experts use "Attack Libraries" to find ways a hacker might hurt a computer system. Some people think systems like STRIDE (a popular way to group threats) are too general or "high level." Because of this, they prefer a more detailed list of specific things that can go wrong.

### What is an Attack Library?

Imagine a real library, but instead of storybooks, it holds files on every way a "bad actor" (a hacker) can attack. These libraries can contain:

- **Proof-of-Concept Code:** Simple code written just to show that an attack is possible.
    
- **Weaponized Exploit Code:** Finished, ready-to-use digital tools that can actually break into a system.
    

### How to Choose a Library

Not all libraries are the same. You have to think about:

1. **Audience:** Who is reading it? A library for experts doesn't need to explain the basics.
    
2. **Detail vs. Abstraction:** * **Abstraction** means looking at the "big picture" (like saying "someone might steal data").
    
    - **Detail** means looking at the exact steps (like saying "someone will use this specific command to take this specific file").
        
3. **Scope:** This is the boundary. A library about website attacks shouldn't have info about office building locks.
    

### Libraries vs. Checklists

- **Checklists:** These are great for making sure you don't forget simple, common mistakes. However, they don't help you think creatively. If a threat isn't on the list, you might miss it.
    
- **Libraries:** These are designed to make you think. They give you enough info to spark new ideas about how your specific system might be at risk.
    

### Two Famous Examples

1. **CAPEC (The Giant Dictionary):** This is a huge, very organized list of almost 500 different attack patterns. It covers everything from physical security to social engineering (tricking people). It is very detailed but can be overwhelming because it's so big.
    
2. **OWASP Top Ten (The "Hits" List):** This is a list of the 10 most dangerous risks for websites. It is updated every year. It is shorter and easier to use than CAPEC if you are building a website.
    

### Technical Terms Defined

- **Threat Modeling:** The process of sitting down and figuring out all the ways someone could attack your app or computer.
    
- **Taxonomy:** A fancy word for a "sorting system." It’s how you group similar attacks together (like putting all "password attacks" in one folder).
    
- **Social Engineering:** Tricking a human being into giving up a password or secret instead of using code to break in.
    
- **Literature Review:** Looking at old reports and news to see what happened to other systems in the past.
