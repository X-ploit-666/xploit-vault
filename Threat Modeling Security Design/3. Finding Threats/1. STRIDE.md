### What is STRIDE?

STRIDE is a helpful word used to remember six specific ways that software can be attacked. It was created by Loren Kohnfelder and Praerit Garg to help people who build software find security problems. Finding these problems is often called "finding threats" or "threat analysis."

### The Six Parts of STRIDE

**1. Spoofing (Pretending to be someone else)**

- **What it is:** This is when a person or a computer program lies about who they are. They might pretend to be a famous person, a specific website, or even a file on your computer.
    
- **Simple Definition:** **Authentication** is the security goal here. Authentication means "proving you are who you say you are." Spoofing breaks this goal.
    
- **Examples:** Someone sending an email that looks like it is from your bank, or a computer program pretending to be a regular system file to trick your computer.
    

**2. Tampering (Changing things without permission)**

- **What it is:** This involves changing information on a disk, a network, or inside a computer's memory.
    
- **Simple Definition:** **Integrity** is the security goal here. Integrity means "making sure data stays correct and hasn't been messed with." Tampering breaks this goal.
    
- **Examples:** Changing the numbers in a spreadsheet, editing a database, or changing the code of a program while it is running.
    

**3. Repudiation (Claiming you didn't do it)**

- **What it is:** This is when someone does something (like buying an item or hitting a button) and then later says, "I didn't do that."
    
- **Simple Definition:** **Non-repudiation** is the security goal here. This means having proof that an action actually happened so no one can lie about it later.
    
- **Examples:** A user claiming they never ordered an expensive item, or a system having no logs (records) to prove who deleted a file.
    

**4. Information Disclosure (Leaking secrets)**

- **What it is:** This is when information is shown to people who aren't allowed to see it.
    
- **Simple Definition:** **Confidentiality** is the security goal here. Confidentiality means "keeping secrets secret." Information Disclosure breaks this goal.
    
- **Examples:** A hacker reading your private emails, or a computer showing your password in an error message by mistake.
    

**5. Denial of Service (Breaking the service)**

- **What it is:** This is an attack that uses up all of a computer's resources (like memory or internet speed) so that the system stops working for everyone else.
    
- **Simple Definition:** **Availability** is the security goal here. Availability means "the system is up and running when you need it." Denial of Service breaks this goal.
    
- **Examples:** Flooding a website with so much fake traffic that real users can't get in, or making a program use up all the computer's memory so it crashes.
    

**6. Elevation of Privilege (Gaining extra power)**

- **What it is:** This is when a regular user finds a way to get the powers of an administrator (a "boss" user).
    
- **Simple Definition:** **Authorization** is the security goal here. Authorization means "only letting people do what they are allowed to do." Elevation of Privilege breaks this goal.
    
- **Examples:** A person with a basic account finding a "bug" (a mistake in the code) that lets them delete other people's files or change system settings.
    

### Code Extraction & Analysis

The text mentions a specific piece of logic (a "fork bomb") as an example of a **Denial of Service** attack:

**The Code:** `while(1){fork();}`

**How it works:**

- **`while(1)`**: This creates an "infinite loop." It tells the computer to keep doing the next step forever and never stop.
    
- **`fork()`**: This is a command that tells the computer to "clone" or "copy" the current program. Every time this runs, one program becomes two.
    
- **The Result**: Because it is inside an infinite loop, the computer will create copies of the program over and over as fast as it can. Very quickly, there will be thousands of programs running. This uses up all the computer's "processing power" and "memory," causing the entire system to freeze or crash.
