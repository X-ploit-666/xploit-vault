Threat Modeling

Finding Security Problems Before They Happen

What is it?

Threat modeling is a way for anyone to find security problems in a system by looking at the "big picture" of how something works instead of zooming into every tiny line of computer code. Think of it like looking at a map of a house to find where a thief might get in, rather than analyzing the type of wood used in the front door.

The STRIDE Model

STRIDE is a checklist used during Step 2 (Brainstorming). It helps you remember the six primary categories of "bad things" that can happen to your system. By evaluating your software against each of these six vectors, you ensure comprehensive threat coverage.

  • S
    Spoofing

    Pretending to be someone else (e.g., using a fake ID).

  • T
    Tampering

    Changing data you aren't supposed to touch.

  • R
    Repudiation

    Doing something bad and claiming you didn't do it.

  • I
    Information Disclosure

    Private secrets leaking to unauthorized people.

  • D
    Denial of Service

    Breaking a system so real users can't use it.

  • E
    Elevation of Privilege

    Getting "Super User" powers you shouldn't have.

Equal assessment across all 6 STRIDE vectors is critical.

Validating Threats

How do you know you did a good job? Once you are done, you should be able to answer "Yes" to these four crucial questions to complete the validation loop.

✔
1. Did we write down or file a bug for every threat we found?
✔
2. Do we have a proposed plan to fix every threat?
✔
3. Did we create a test case to see if the fix works?
✔
4. Did the software pass that test?

Anyone can learn to threat model. It helps developers build better things, helps administrators set them up safely, and helps security experts teach others how to stay protected.
