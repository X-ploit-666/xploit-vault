> 01_THE_STYLING_ENGINE

If HTML is the bare skeleton of a webpage, CSS (Cascading Style Sheets) is the skin, clothing, and makeup. It is a special coding language used to design and format everything on a web page. Just like apps on your phone get updates, CSS gets new versions over time. These updates bring cool new design tools, and web browsers automatically update so they can understand and display these new features.

At its core, CSS works by finding specific parts of your HTML—like the main background (body) or big titles (h1)—and giving them visual rules. You can change almost anything: the type of text, the size, the background color, and exactly where things sit on the screen. Remember those "class" and "id" name badges we put on HTML tags? CSS uses those exact badges to find specific parts of the page and paint them.

> SYNTAX_AND_RULES

CSS has a very simple rulebook. First, you type the name of the HTML part you want to change. Next, you open curly brackets { }. Inside those brackets, you list your design choices. Every choice has two parts: a Property (what you want to change, like "color") and a Value (what you are changing it to, like "red").

There are hundreds of properties you can play with, such as height, borders, spacing (margin and padding), and text alignment. By mixing and matching these rules, you can make a webpage look amazing.

CSS is not just for still pictures. It is powerful enough to make things move! You can use special CSS commands (like @keyframes) to create smooth animations, moving items across the screen, or even building 3D effects.

CSS is a team player. It often works alongside JavaScript (the code that handles actions). Together, they can make a website react dynamically. For example, a picture might tilt or change colors based on exactly where your mouse pointer is moving. CSS is also used outside of normal web pages—it helps design mobile phone apps and other data formats like XML and SVG graphics.

> 02_CSS_FRAMEWORKS

Writing every single design rule from scratch for hundreds of buttons and boxes can be exhausting and slow. To solve this, developers created CSS Frameworks.

Think of a framework like a massive, pre-built toolbox full of beautiful, ready-to-use design templates. Instead of spending hours writing code to make a button look perfectly round and shiny, you simply borrow the code from the framework. These toolboxes are heavily optimized, work perfectly with JavaScript, and contain everything needed to build a modern app quickly. Some of the most famous toolboxes (frameworks) include:

  • Bootstrap
  • SASS
  • Foundation
  • Bulma
  • Pure

> 03_CODE_ANALYSIS

Here is a basic example of CSS code. Below, we break down exactly what each rule does to the HTML page.

body {
  background-color: black;
}

h1 {
  color: white;
  text-align: center;
}

p {
  font-family: helvetica;
  font-size: 10px;
}
Code Line / Block What it does in simple terms
body { ... } This targets the main visible background area of the entire webpage. Everything inside the curly brackets will happen to the background.
background-color: black; This is the rule inside the body. It tells the browser: "Paint the entire background of the website solid black."
h1 { ... } This targets all the large, main headings (Titles) on the webpage.
color: white; This changes the color of the text for the headings to white (so they can be seen on the black background).
text-align: center; This pushes the heading text to sit exactly in the middle of the screen, instead of staying on the left side.
p { ... } This targets all the normal, regular text paragraphs on the webpage.
font-family: helvetica; This changes the style (the font type) of the paragraph text to a clean font named "Helvetica".
font-size: 10px; This sets the size of the paragraph text to 10 pixels, which will make the text look quite small.
