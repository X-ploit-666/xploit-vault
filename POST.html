The Power of POST

Unlike GET requests which expose parameters in the URL, the POST method encapsulates data within the request body. This fundamental difference enables the transmission of sensitive credentials, large files, and complex binary data without leaving traces in browser history or server access logs.

The Digital Hall Pass: Cookies & Sessions

HTTP is stateless; it has "short-term memory loss." To remember a user after login, the server issues a Session Cookie (e.g., PHPSESSID). This token acts as a digital keycard, granting access to restricted areas without requiring re-authentication for every click.

Speaking the Language: JSON

Modern APIs and Single Page Applications rarely use standard form encoding. Instead, they utilize JSON (JavaScript Object Notation). When sending JSON data, the server must be explicitly informed via the Content-Type: application/json header, or the request may be ignored.

The Hacker's Toolkit: Crafting Requests

Security professionals bypass the browser UI to interact directly with the server. Using cURL, we can simulate logins, inject headers, and hijack sessions.

# 1. Manual Login via POST

$ curl -X POST
  -d 'user=admin&pass=12345'
