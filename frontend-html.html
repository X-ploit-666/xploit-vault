> 01_THE_BASICS

HTML (HyperText Markup Language) is the invisible building block of every website. Think of it as the skeleton of a web page. It holds the basic parts you see online, like titles, text boxes to type in, and pictures. Your web browser (like Chrome or Safari) acts as a translator. It reads this invisible skeleton and builds the visible page on your screen.

HTML is built like a family tree. At the very top of this tree is the Document. Inside the Document is the main HTML container. This main container tells the computer, "This is a web page," which is different from other data formats (like XML, which is just another way computers organize data). Inside this main container, you can put smaller parts, and those smaller parts can hold even smaller parts.

Every part of the page is called an Element. Elements are created using "tags." A tag is just a word wrapped in angle brackets, like <p> for a paragraph. Most elements have an opening tag to start it and a closing tag to end it. The actual text you want to show goes in the middle. Tags can also have special name badges called "id" or "class" (for example, <p id='red-paragraphs'>). These name badges help CSS (the code that adds colors and design) know exactly which part of the page to paint. An element is the whole package: the opening tag, the content, and the closing tag.

> URL_ENCODING_PROTOCOL

A very important rule for web pages is called URL Encoding (or percent-encoding). Web addresses (URLs) can only use a basic alphabet called ASCII (simple English letters, numbers, and a few symbols). If you need to use a special symbol in a web link, you must translate it into a secret code so the browser doesn't get confused.

This secret code always starts with a percent sign (%) followed by two letters or numbers. For example, a single quote (') changes into %27. Also, web addresses are not allowed to have blank spaces! A space must be changed into a plus sign (+) or %20. Security testers use special tools, like a popular program called Burp Suite, to easily translate these codes back and forth.

A web page is split into two main sections. The <head> section holds hidden settings, like the title you see on your browser tab. The <body> section holds everything you can actually see on the screen. There are other hidden parts too, like <style> for adding colors (CSS) and <script> for adding interactive actions (JavaScript).

Each of these pieces on the page is part of a system called the DOM (Document Object Model). The DOM is basically a digital map created by the web standard group (W3C). It allows other programs to change the text, look, or structure of the page while you are looking at it. The DOM rulebook has three parts: Core DOM (for any data), XML DOM (for XML data), and HTML DOM (specifically for web pages).

Because the DOM is a map (like document.head), it helps us find exactly where every piece of the page lives. We can search for these pieces using their special "id" or "class" names. Understanding this map is very important for cybersecurity. If a website is poorly built, a hacker can use a trick called XSS (Cross-Site Scripting) to sneak their own dangerous elements onto the page and change how it works.

> 02_CODE_ANALYSIS

Here is a very basic example of HTML code. Below, we break down exactly what each line does.

<!DOCTYPE html>
<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <h1>A Heading</h1>
        <p>A Paragraph</p>
    </body>
</html>
Line of Code What it does in simple terms
<!DOCTYPE html> This is an instruction to the web browser. It says, "Hello, this file is written in modern HTML." It makes sure the page loads correctly.
<html> This is the master wrapper. It opens the main HTML container. Everything else on the web page goes inside this wrapper.
<head> This opens the hidden settings section. Information here is used by the browser but isn't directly shown on the main screen.
<title>Page Title</title> This sets the name of the webpage. The text "Page Title" is what you will see on the tab at the very top of your web browser.
</head> This closes the hidden settings section. (The slash / means "end here").
<body> This opens the visible section. Everything you read, click on, or look at on the screen goes inside here.
<h1>A Heading</h1> This creates a large, bold title on the screen. The "h" stands for heading, and "1" means it is the biggest, most important size.
<p>A Paragraph</p> This creates a normal block of text. The "p" stands for paragraph.
</body> This closes the visible section of the webpage.
</html> This closes the master wrapper, marking the absolute end of the web page.
